# Camera Package

The camera package handles frame acquisition from USB webcams and the ZED 2i stereo camera, runs YOLOv8 object detection on each frame, and exposes an MJPEG streaming server so any browser on the network can watch live annotated feeds.

---

| File | Role |

## Camera backends

### USB camera (`UsbCamera`)

Uses OpenCV `VideoCapture`. On Linux it reads `/sys/class/video4linux/videoN/name` to skip device nodes whose name contains `"zed"`, preventing the USB camera thread from accidentally grabbing the ZED 2i's virtual video device. The starting device index is controlled by `USB_CAMERA_INDEX`.

### ZED 2i (`ZedCamera`)

Uses the ZED SDK (`pyzed`). Opens at 720p, `PERFORMANCE` depth mode, metric units, 20 m max depth. For every detected object the centre pixel is queried for depth so a distance (metres) is stored alongside the bounding box.

If `pyzed` is not installed, the ZED thread logs an error and exits immediately ‚Äî the rest of the package continues normally.

---

## Object detection (`ObjectDetector`)

Wraps `ultralytics.YOLO`. Bounding boxes are normalised to `[0, 1]` relative to the frame dimensions so they are resolution-independent in the database.

If `ultralytics` is not installed, `detect()` returns an empty list and a warning is logged.

### `Detection` dataclass

| Field | Type | Description |
|---|---|---|
| `class_name` | `str` | YOLO class label |
| `confidence` | `float` | Detection confidence (0‚Äì1) |
| `bbox_x` | `float` | Normalised top-left x |
| `bbox_y` | `float` | Normalised top-left y |
| `bbox_w` | `float` | Normalised width |
| `bbox_h` | `float` | Normalised height |

## Custom model training

The competition uses six RoboSub emoji targets. A custom YOLOv8 model is trained from synthetic data generated by `libs/camera_package/emoji_data_generator.py`.

### Classes

| ID | Name | Expected range |
|---|---|---|
| 0 | `compass` | ~3 m |
| 1 | `hammers` | ~2 m |
| 2 | `life_preserver` | ~2 m |
| 3 | `sos` | ~1 m |
| 4 | `fire` | ~1 m |
| 5 | `blood_drop` | ~1 m |

### Training pipeline

`emoji_data_generator.py` runs five steps end-to-end:

| Step | CLI flag | What it does |
|---|---|---|
| 1 ‚Äî Extract | `--generate` / `--all` | Renders each PDF page to a high-res RGBA PNG (one emoji per page, 300 dpi) |
| 2 ‚Äî Crawl | `--crawl` / `--all` | Downloads underwater background images via Bing; falls back to procedural gradients |
| 3 ‚Äî Generate | `--generate` / `--all` | Composites emojis onto backgrounds with alpha-blend, rotation, and IoU-filtered placement |
| 4 ‚Äî Config | `--generate` / `--all` | Writes `dataset.yaml` for the YOLOv8 trainer |
| 5 ‚Äî Train | `--train` / `--all` | Trains `yolov8s.pt` (100 epochs, 640 px, batch 32, `multi_scale=True`) |

**End-to-end example (run on the AI server):**

```bash
cd libs/camera_package
python emoji_data_generator.py --pdf '/path/to/Robosub Emojies.pdf' --all
```

**Generate for long range only:**

```bash
python emoji_data_generator.py --generate --range far
```

**Fine-tune from a previous checkpoint:**

```bash
python emoji_data_generator.py --train --base-model model_training/runs/v2/best.pt
```

### Range profiles

| Profile | `--range` | Emoji scale | Hard-negative ratio | Notes |
|---|---|---|---|---|
| Close | `close` | 0.02 ‚Äì 0.35 | 10% | 1‚Äì3 m |
| Far | `far` | 0.01 ‚Äì 0.06 | 15% | 6‚Äì7 m; heavier blur/fog/noise augmentations |

Pass `--range all` to generate both datasets sequentially.

### Augmentation pipeline

Both profiles apply: random brightness/contrast, Gaussian blur, motion blur, Gaussian noise, hue-saturation shift, random fog, random shadow, JPEG compression, and ¬±15¬∞ rotation. The `far` profile uses wider kernels and higher probabilities for each blur/noise/fog stage to simulate water-column degradation.

### Model versions

Trained weights live in `libs/camera_package/model_training/runs/`:

| Version | Path | Base model | Notes |
|---|---|---|---|
| v1 | `runs/v1/best.pt` | `yolov8n.pt` | Initial baseline |
| v2 | `runs/v2/best.pt` | `yolov8s.pt` | Switched to small architecture; added class weighting (hammers √ó3, fire/blood_drop √ó2.5); 100 epochs |
| v3 | `runs/v3/best.pt` | `runs/v2/best.pt` | Fine-tuned from v2; added far-range dataset and improved augmentations |

### Using a trained model

Point `YOLO_MODEL` at a `best.pt` in `.env`:

```env
YOLO_MODEL=libs/camera_package/model_training/runs/v3/best.pt
YOLO_CONF=0.5
```

The path is resolved relative to the repo root when the camera service starts.

## Streaming server

A minimal FastAPI app served by uvicorn on `CAMERA_STREAM_HOST:CAMERA_STREAM_PORT`.

| Route | Description |
|---|---|
| `GET /` | HTML dashboard showing both feeds side-by-side |
| `GET /stream/zed` | MJPEG stream from the ZED 2i |
| `GET /stream/usb` | MJPEG stream from the USB webcam |

Streams run at ~30 fps (`asyncio.sleep(0.033)`). If a camera has not produced a frame yet, a grey `NO SIGNAL` placeholder JPEG is sent instead.

---

## Database integration

Each detection is posted to the DB Manager's `detections` table via `AUVClient`. The `CAMERA` field is `"zed"` or `"usb"`. `DISTANCE` is `-1.0` for USB detections (no depth sensor).

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `ZED_CAMERA` | `false` | Set `true` to enable the ZED 2i thread |
| `USB_CAMERA` | `false` | Set `true` to enable the USB webcam thread |
| `USB_CAMERA_INDEX` | `0` | `/dev/videoN` index to start scanning from |
| `CAMERA_STREAM_HOST` | `0.0.0.0` | Bind address for the MJPEG server |
| `CAMERA_STREAM_PORT` | `8001` | Port for the MJPEG server |
| `YOLO_MODEL` | `yolov8n.pt` | Path to a YOLOv8 `.pt` weights file; use a `model_training/runs/vN/best.pt` for the custom emoji model |
| `YOLO_CONF` | `0.5` | Minimum confidence threshold for detections |

## Running standalone

```bash
# enable at least one camera in .env first, e.g.:
# ZED_CAMERA=false
# USB_CAMERA=true
# USB_CAMERA_INDEX=0

uv run python -c "from libs.camera_package.camera_manager import run; run()"
```

Or via `ProcessManager` / the TUI (service name `"camera"`):

```python
from libs.process_manager import ProcessManager
pm = ProcessManager()
pm.start("camera")
```

Live feeds at `http://localhost:8001` once the service is running.
